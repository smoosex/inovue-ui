Default to using Bun instead of npm/yarn/pnpm.

- Use `bun install` instead of `npm install` or `yarn install` or `pnpm install`
- Use `bun run <script>` instead of `npm run <script>` or `yarn run <script>` or `pnpm run <script>`
- Use `bunx <package> <command>` instead of `npx <package> <command>`

Always use `type` instead of `interface` for type definitions.

Never change `src/lib/utils.ts` and `src/components/ui/**/*` unless the user requests, that is generated by shadcn-vue cli.

Always use Kebab-case for Event Names.

## Commands

```bash
# Development
bun run dev

# Build for production
bun run build

```

## Architecture

### Tech Stack

- **Vue 3** with Composition API and `<script setup>`
- **TypeScript** with strict mode
- **Tailwind CSS v4** for styling
- **Pinia** for state management (with persisted state plugin)
- **Vue Router** for routing
- **Axios** for HTTP requests
- **MSW** for API mocking in development

### Directory Structure

```
src/
├── api/              # Shared API functions
├── assets/           # Static assets (icons in subfolders)
├── components/
│   ├── ui/           # shadcn-vue style primitive components
│   └── vue-bits/     # Custom business components
├── features/         # Feature-based modules
│   └── {feature}/
│       ├── api.ts
│       ├── components/
│       ├── composables/
│       ├── types.ts
│       └── index.ts
├── layouts/          # Layout wrappers (ClassicLayout, BoldLayout, ModernLayout)
│   └── components/   # Layout-specific components
├── locales/          # i18n messages (en/, zh-hans/)
├── mocks/            # Mock data
├── plugins/          # Vue plugins (i18n, pinia, router registration)
├── router/           # Router configuration and routes
├── stores/           # Pinia stores (theme, auth, menu)
├── styles/           # Global styles
├── types/            # TypeScript types and auto-import configs
├── utils/            # Utilities (http client wrapper)
└── views/            # Views (pages)
```

### Key Patterns

**Features**: Business logic organized by feature. Each feature has `api.ts`, `components/`, `composables/`, `types.ts`, and `index.ts`.

**API Layer**: Use `src/utils/http.ts` wrapper for API calls. Define response types in `src/types/response.ts`. API functions are exported from `features/{feature}/api.ts`.

**Stores**: Pinia stores use setup store syntax with `persist: true` for persistence.

**Components**: UI primitives in `components/ui/` are add by shadcn-vue cli. Import directly by component name (auto-imported).

**Routing**: Routes are split across files in `router/routes/`.

**i18n**: Messages are loaded in `src/plugins/i18n.ts`.

### Theming

**Themes**: Multiple color themes (amethyst, bubblegum, catppuccin, claude). Theme is managed by `useThemeStore` and applied via CSS class prefixes on `document.documentElement`. Theme files are located in `src/assets/styles/themes/`.

**Layouts**: Two layouts (classic, frosted) defined in `src/layouts/`. Layout-specific component styles are overridden via CSS in `src/assets/styles/layouts/`.

### Environment Variables

| Variable             | Description                     |
| -------------------- | ------------------------------- |
| VITE_API_PREFIX      | API base path                   |
| VITE_API_MSW         | Enable MSW mocking (true/false) |
| VITE_API_PROXY       | Enable API proxy                |
| VITE_API_TARGET      | Proxy target server             |
| VITE_API_REPLACEMENT | Proxy replacement prefix        |
